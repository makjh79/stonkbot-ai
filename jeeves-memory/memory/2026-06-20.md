# 2026-06-20 — StonkBOT.AI UI/UX Iteration Session

## What we did
Iterated the StonkBOT.AI website UI with rapid deploys from v33 to v44.

**Major changes:**
- v33: increased data density (tighter padding, smaller gaps, compact hero).
- v34: aurora gradient behind header, hover-lift on cards, bottom-nav active cyan dot, gradient section-header dividers.
- v35–v39: introduced centered 112px sparklines in holdings rows; removed the 💡 hint icon.
- v40: wired up a 30-day price-history backend (`scripts/fetch_price_history.py`) with `website/history/` JSON files and daily cron at 06:30 HK. Currently synthetic (avg_entry → current) because Yahoo Finance and Finnhub candles are blocked/paid on the server.
- v41: renamed section header "The $100K Experiment" → "Bot vs. Market".
- v42: removed "Tap for allocation details" under Sectors.
- v43: renamed FAQ header "Questions?" → "FAQ"; made race cards tappable with hover lift; added sector-row hover highlight; added faint gradient separator between hero and first section.
- v44: smoother FAQ expand/collapse animation; section titles uppercase with letter-spacing for premium feel.

## Key fixes / ops
- Discovered GitHub Actions deploys were silently failing because an untracked `scripts/fetch_price_history.py` on the server blocked `git pull`. Manually pulled and cleared it; deploys are clean again.
- Cloudflare + origin now serve the correct HTML.
- User's earlier "desktop cache/buttons not updating" issue was partly real deploy failures, not purely browser cache.

## User preferences reinforced
- Highly clutter-sensitive; prefers minimal indicators and terse copy.
- Likes subtle motion and polish, not flashy.
- Sparklines should be centered, wider, and meaningful; user was okay with synthetic history as a pragmatic fallback.
- Section copy: "Bot vs. Market" was preferred over battle/war clichés.
