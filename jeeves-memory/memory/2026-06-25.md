## Stonkbot VPS Health Alert — Signal Update Service Fix

**Alert:** Health monitor flagged DEGRADED — all watchlist positions in TRACKING, no scored signals. `stonk-ai-signal-update.service` was failed.

**Root cause:** `update_signal_performance.py` and `export_signals_for_website.py` imported module-level functions (`load_signal_history`, `save_signal_history`, `get_signal_stats`, `export_for_website`) from `signal_tracker.py` — but those functions didn't exist at module level. `signal_tracker.py` only had them as class methods on `SignalTracker` and two standalone functions (`log_buy_signal`, `update_signal_performance`).

**Secondary issue:** `signals.json` is written by the signal engine with a different schema (momentum/quality scores, no `status`/`outcome` fields) than what `SignalTracker` expects (tracked signals with `status`, `outcome`, `entry_price`). This caused `KeyError: 'status'` when the tracker methods iterated over all signals.

**Fix applied:**
1. Added compatibility wrapper functions to `signal_tracker.py`: `load_signal_history()`, `save_signal_history()`, `get_signal_stats()`, `export_for_website()`
2. Made `SignalTracker` class methods robust to mixed-format signals (using `.get('status')` instead of `['status']`, filtering to only tracked signals)
3. Re-enabled the service and its 5-minute timer (`stonk-ai-signal-update.timer`)

**Note:** The "all TRACKING" alert was somewhat stale — the main trading bot (`stonk-ai.service`) was actually running fine and generating 11 signal candidates (AMAT, AFRM, AMD in NOW tier). The signal-update service just couldn't track performance because of the import errors.
