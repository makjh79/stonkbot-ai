## Signals Stale Alert (08:00 UTC)
- **Problem:** `signals.json` was 500+ minutes stale. Health check flagged DEGRADED.
- **Root cause:** `trading_bot.py` only called `maybe_refresh_signals()` inside `run_cycle()`, which early-exits when market is closed. So signals froze after last market-close cycle.
- **Also:** `signal_engine.py` cron was only daily at 12:30 UTC — not enough redundancy.

1. **Moved `maybe_refresh_signals()` above `run_cycle()`** in main loop so it refreshes even outside market hours. Now signals.json updates every 15 min regardless of market state.
2. **Fixed bottom guard patches** — 403 duplicate-sell guard and market-order sell wrapper were sitting *after* `if __name__ == "__main__"`, so they never loaded. Moved above it.
3. **Added `exc_info=True`** to `_execute_sell` error logging so future sell failures produce full stack traces.
4. **Restarted stonk-ai.service** — bot came up clean, generated fresh signals at 08:19 UTC and again at 10:25 UTC.

## Extended Hours Data Fix (11:50 UTC)
- **Problem:** Holdings popups had zero extended-hours data. Watchlist popups had blank after-hours lines in the morning.
- **Holdings:** `generate_popup_content.py` never fetched extended-hours bars. Fixed by copying pre-market/after-hours fields from `ai_watchlist_live.json` into popup narratives.
- **Watchlist:** `get_extended_hours_bars()` only queried *today's* 20:00-23:59 UTC window. At 7am ET, that window hasn't opened → always blank. Fixed by pulling yesterday's after-hours bars when current time is before 8pm UTC.
- **Result:** Watchlist now shows 7/20 after-hours and 9/20 pre-market. Holdings show 5/11 with both fields.

## Design Decision: No Synthetic After-Hours Data
- Howie asked why fintech apps show after-hours data for all stocks while StonkBOT doesn't.
- **Reason:** Alpaca SIP only returns bars for actual trades. Zero volume = no bar.
- **Fintech apps** use premium data aggregators (Polygon paid tier, Refinitiv, IEX) that fill gaps with interpolated/synthetic data.
- **Recommendation:** Keep honest blanks. Don't add synthetic "0.00%" lines. If UX needs polish, show `—` or hide the field when missing.
- **Consensus:** Stick with zero external dependencies. Don't wire in another data provider just for cosmetic after-hours lines.
- **Exception:** If we ever want true fintech-level coverage, options are Polygon Stocks Starter, IEX Cloud, or Tiingo. But adds cost, keys, and rate-limit risk.
