## Dynamic Watchlist Sync (Session Continuation)

### Problem Identified
User correctly pointed out that AI Council sentiment wasn't dynamically syncing with the watchlist. Both `fetch_ai_watchlist.py` and `fetch_crowd_sentiment.py` had hardcoded symbol lists that could drift out of sync.

### Solution Implemented

**Single Source of Truth:**
- `ai_watchlist_live.json` = Master symbol list
- Both fetchers now read symbols dynamically from this file

**Changes Made:**

1. **fetch_ai_watchlist.py:**
   - Added `get_watchlist_symbols()` function
   - Reads from `ai_watchlist_live.json` on each iteration
   - Falls back to `DEFAULT_WATCHLIST` if file not found
   - Updated `fetch_watchlist_prices()` to use dynamic symbols
   - Updated `main()` to refresh symbols each loop
   - Updated `council_autonomous_watchlist_adjustment()` to use dynamic base

2. **fetch_crowd_sentiment.py:**
   - Enhanced `get_watchlist_symbols()` to try multiple paths
   - Tries `/opt/stonk-ai/` first, then `/var/www/hedge-fund-website/`
   - Same fallback list for consistency

**Sync Architecture:**
```
┌─────────────────────────┐
│ ai_watchlist_live.json  │ ← SOURCE OF TRUTH
│ (master symbol list)    │
└───────────┬─────────────┘
            │
    ┌───────┴───────┐
    │               │
    ▼               ▼
┌──────────┐   ┌──────────────┐
│ AI Fetch │   │ Crowd Fetch  │
│ (prices) │   │ (sentiment)  │
└────┬─────┘   └──────┬───────┘
     │                │
     └───────┬────────┘
             ▼
┌─────────────────────────┐
│     Website Display     │
│ (Social vs AI Battle)   │
└─────────────────────────┘
```

### Result
- Adding/removing stocks from watchlist → Both AI and Crowd sentiment auto-update
- No more manual syncing required
- 5-minute cron interval keeps everything fresh

## Price Accuracy Crisis & Resolution (Session Continuation)

### Problem Identified
User discovered watchlist prices were significantly incorrect compared to market:
- AMZN showing $226.10 (actual: $238.56) - **$12.46 off!**
- NFLX showing $75.89 (actual: $80.33) - **$4.44 off!**
- COIN showing $168.97 (actual: $159.72) - **$9.25 off!**

### Root Cause
- **Old code** used Alpaca's `/quotes/latest` endpoint
- Quotes endpoint returns **bid/ask spreads**, not actual traded prices
- During off-hours, bid/ask can be stale while trades show true price

### Solution Implemented

**Changed endpoint:**
```python
# OLD (WRONG):
resp = requests.get(f"{base}/v2/stocks/quotes/latest?symbols={symbols_str}")

# NEW (CORRECT):
resp = requests.get(f"{base}/v2/stocks/trades/latest?symbols={symbols_str}")
```

**Price Validation Layer:**
- Added `verify_prices.py` - Daily Yahoo Finance comparison
- 50 cent tolerance threshold
- Logs discrepancies to `price_verification.json`
- Runs daily at 6 PM via systemd timer

**Safeguards Added:**
1. Trades endpoint (not quotes) for accuracy
2. Price validation (must be > $0)
3. Sanity check (warn if > 20% change from previous)
4. Staleness detection (warn if trade > 24h old)
5. Auto-logging of all discrepancies

### Verification Results
All 12 watchlist prices now verified against Yahoo Finance:
- AMZN: $238.56 ✅ (matches Yahoo)
- NFLX: $80.33 ✅ (matches Yahoo)
- COIN: $159.72 ✅ (matches Yahoo)
- All others: Within $0.50 tolerance ✅

**Report:** `/var/www/hedge-fund-website/price_verification.json`

### New Service Created

## UI Bug Fix - Misleading Price Display (Session Continuation)

### Problem Reported
User screenshot showed: **AMZN $238.56 → $358 (+50.0%)**

This looked like AMZN was up 50% today, which was clearly wrong!

### Root Cause
In `renderAIWatchlist()` function, the price card was displaying:
```javascript
intrinsic: price * 1.5, // Estimate intrinsic as 50% upside
upside: '+50.0%',
// Displayed as: $238.56 → $358 (+50.0%)
```

The code was showing our **+50% profit exit target** as if it was the stock's actual daily performance. This was confusing and misleading.

### Fix Applied

**Changed display from:**
```javascript
<span style="font-size: 13px; color: ${upsideColor}; font-weight: 700;">→ $${stock.intrinsic.toFixed(0)} (${upsideStr})</span>
```

**To actual daily change:**
```javascript
<span style="font-size: 13px; color: ${changeColor}; font-weight: 700;">${changeStr}</span>
```

Where `changeStr` is calculated from real `change_pct` data from Alpaca.

### Result
- Card now shows: **$238.56 +0.00%** (correct for weekend)
- No more misleading "→ $358 (+50.0%)" text
- The +50% target still exists in the detailed stock modal (properly labeled as "Target Price")
- Weekend prices correctly show 0% change (market closed)
